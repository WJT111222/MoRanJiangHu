# Bugfix Requirements Document

## Introduction

本 bugfix 是一批由用户反馈的综合缺陷，覆盖三个模块：背包物品图片查看器、地图系统（层级导航 / NPC 位置 / 建筑-道路布局 / 野外场景 / NPC 与社交面板一致性）、以及主题背景颜色持久化。这些缺陷有的影响核心交互（点击物品图片后无法关闭、主题反复丢失），有的严重影响沉浸感和世界观一致性（NPC 无位置被随机乱排、青石镇建筑重叠道路穿墙、野外出现城镇式路网等）。为便于按独立 bug 条件 C(X) 验证修复与回归预防，本文档把九条反馈拆成九条独立子 bug，每条都给出触发条件、错误行为、期望行为，以及必须保持不变的相邻行为。

## Bug Analysis

### Current Behavior (Defect)

背包物品图片放大查看器：

1.1 WHEN 玩家在背包中点击一件物品的图片以放大查看 THEN 系统显示放大图但未在图片右上角提供醒目的关闭按钮，玩家找不到退出入口，只能通过其他手势或刷新页面退出
1.2 WHEN 放大的物品图片展示给玩家 THEN 图片本身不是写实风格（当前使用的是偏卡通/插画或风格不统一的底图）
1.3 WHEN 放大的物品图片展示给玩家 THEN 图上同时出现模型或生成服务附加的水印以及物品名/描述文字，叠加在图片像素之上

地图系统：

1.4 WHEN 地图 AI 生成流程为某一层级产出 NPC 列表，且其中任意 NPC 缺少坐标字段（x/y 未提供或为 null） THEN 系统回退到"随机散点"逻辑把这些 NPC 随机摆放在地图上，忽略 AI 剧情里的实际位置语义
1.5 WHEN 玩家进入最内层层级（例如"荒庙内部"这种建筑内部视图） THEN 系统仍按城镇/野外层级的方式生成道路（roads），导致建筑内部出现"路"，而不是仅由墙、门、房间组成
1.6 WHEN 玩家在中原 / 青石镇 / 郊外 / 荒庙这类有层级的地图上，需要切换已展开过的上级层级 THEN 系统不把这些层级节点本身做成可点击选中，而是额外显示"返回上一级：郊外"这类按钮作为唯一切换入口
1.7 WHEN 地图上生成 NPC 列表 THEN 系统会生成并展示在社交面板角色名单中不存在的 NPC（地图独有幽灵 NPC），或反之，社交面板里的角色从未出现在任何层级地图上，两个子系统数据不对齐
1.8 WHEN 玩家进入"郊外-荒庙"这一类野外层级 THEN 系统按城镇/居住区的路网规则为其生成多条横纵交错的道路网格，与"野外一条小路旁有一座庙"的剧情语义不符
1.9 WHEN 青石镇层级的地图被生成 / 重新生成 THEN 建筑物之间发生几何重叠，且道路走线直接穿越建筑像素（道路和建筑碰撞未被解算）
1.10 WHEN 玩家进入最顶层"中原"层级 THEN 系统把它当作一般层级，仅显示单个城镇或零散要素，而没有"由青石镇等多个城市以道路相连、其间点缀山脉/湖泊/森林等大地物"的区域地图语义

主题背景颜色设置：

1.11 WHEN 玩家在设置里修改"主题/背景颜色"并关闭设置面板后，重新打开设置或刷新页面 THEN 主题颜色恢复为默认值，上次选择没有被持久化

### Expected Behavior (Correct)

背包物品图片放大查看器：

2.1 WHEN 玩家在背包中点击一件物品的图片以放大查看 THEN 系统 SHALL 按 AGENTS.md 中"Image Viewer Close Button Rule"在图片右上角渲染醒目的红色圆形关闭按钮（h-12 w-12，白色描边，红色阴影，hover 放大），点击后关闭放大视图
2.2 WHEN 系统为物品生成 / 展示放大图 THEN 系统 SHALL 通过写实风格提示词（realistic / photo-realistic / highly detailed 等）生成或筛选物品图片，使其与游戏整体写实美术方向一致
2.3 WHEN 系统为物品生成 / 展示放大图 THEN 系统 SHALL 在提示词中明确禁止水印 / logo / signature / 文字标签，并在必要时对返回图做后处理（或重试）以保证最终展示给玩家的放大图里没有模型水印、也没有叠加的物品名/描述文字

地图系统：

2.4 WHEN 地图 AI 生成某层级的 NPC 列表时，遇到任意 NPC 缺少坐标字段 THEN 系统 SHALL 调用 AI 基于该层级的剧情/场景语义（如"掌柜→在柜台附近"、"和尚→在庙内"）补齐合理位置，而不是随机散点摆放
2.5 WHEN 地图进入最内层 / 建筑内部层级 THEN 系统 SHALL 仅按"墙 + 门 + 房间"的组合生成布局，完全不生成任何 road / 道路要素
2.6 WHEN 玩家在有多级层级的地图（例如中原 → 青石镇 → 郊外 → 荒庙）间切换 THEN 系统 SHALL 把当前路径上的每一级节点本身做成可点击区域，选中态以高亮区分，并且 SHALL NOT 依赖额外的"返回上一级：XX"文字按钮作为唯一导航入口（该按钮可完全移除或仅作辅助）
2.7 WHEN 系统为地图生成或显示 NPC 时 THEN 系统 SHALL 保证地图 NPC 集合与社交面板角色集合一致：地图上不出现社交面板不存在的角色，社交面板里每个应出现在当前区域的角色也都能在对应层级地图上看到
2.8 WHEN 玩家进入野外类层级（郊外 / 荒野 / 山道 / 荒庙周围等） THEN 系统 SHALL 通过专门的野外提示词/布局逻辑生成：至多一条主路贯穿地图，少量建筑/地标沿主路两侧分布，不使用城镇横纵网格路网
2.9 WHEN 青石镇这类城镇层级的地图被生成 THEN 系统 SHALL 在布局后进行碰撞解算：任意两栋建筑的包围盒不得重叠，任意一条道路的轨迹不得穿越任何建筑内部；若初次布局冲突应重试或调整位置直到满足
2.10 WHEN 玩家进入最顶层"中原"这一大区层级 THEN 系统 SHALL 以"区域地图"方式呈现：显示多个下级城市节点（如青石镇等），城市之间由道路相连，并按语义点缀山脉 / 湖泊 / 森林等大地物（macro features），而不是只显示单一城镇

主题背景颜色设置：

2.11 WHEN 玩家修改主题 / 背景颜色设置 THEN 系统 SHALL 将新值实时写入 IndexedDB `WuxiaGameDB` 的 `settings` store（或 localStorage，视现有实现），并在应用启动 / 设置面板重新打开 / 页面刷新后从同一存储正确读出该值并应用到界面

### Unchanged Behavior (Regression Prevention)

背包物品图片放大查看器：

3.1 WHEN 玩家在其他模块（社交角色头像放大、装备立绘放大、图像管理器）中打开图片查看器 THEN 系统 SHALL CONTINUE TO 按 AGENTS.md 现有规则在右上角展示一致样式的红色圆形关闭按钮、右对齐定位和 85vw 最大宽度（不被此次修复回退或覆盖）
3.2 WHEN 物品尚未生成图片或使用默认图标 THEN 系统 SHALL CONTINUE TO 按现有的占位/默认图标规则展示，不强制阻塞显示
3.3 WHEN 系统按"Item Image Generation Queue Rule"以单并发生成物品图片 THEN 系统 SHALL CONTINUE TO 维持同一时刻仅有一个物品图片生成任务的节流策略，不因本次提示词/风格修改而并发生成

地图系统：

3.4 WHEN AI 已经为某 NPC 明确提供了合理坐标 THEN 系统 SHALL CONTINUE TO 使用 AI 返回的坐标，不把它强行覆盖为新补齐逻辑生成的位置
3.5 WHEN 地图处于非最内层的外层（城镇、野外、大区） THEN 系统 SHALL CONTINUE TO 按各自现有规则生成道路 / 主路 / 区域路网，不因"建筑内部无道路"这条修复而波及外层
3.6 WHEN 玩家进入仅有单一层级的地图 THEN 系统 SHALL CONTINUE TO 正确展示该层级，不因引入多级可点击 breadcrumb 高亮而破坏单层显示
3.7 WHEN AI 返回的 NPC 列表与社交面板角色列表在某一区域本就完全一致 THEN 系统 SHALL CONTINUE TO 按 AI 返回内容渲染，不做任何额外过滤或增删
3.8 WHEN 玩家进入城镇 / 村庄等居住区层级且建筑数量满足 `utils/mapLayoutOptimizer.ts` 触发阈值（≥4） THEN 系统 SHALL CONTINUE TO 按"Map Layout Optimizer Integration"中描述的路网 + 街区优化逻辑进行布局（仅在野外、最内层和顶层大区走不同分支）
3.9 WHEN 青石镇以外的其他城镇层级已无建筑重叠与路穿建筑问题 THEN 系统 SHALL CONTINUE TO 按现有布局结果呈现，不因青石镇碰撞修复引入新的布局偏移或重新洗牌
3.10 WHEN 中原层级以外的其他层级（青石镇 / 郊外 / 荒庙内部等） THEN 系统 SHALL CONTINUE TO 维持其既有的层级语义和渲染方式，不被"中原大区地图"的变更所影响

主题背景颜色设置：

3.11 WHEN 玩家未修改主题 / 背景颜色，或使用的是默认主题 THEN 系统 SHALL CONTINUE TO 按默认主题正确渲染界面
3.12 WHEN 玩家修改除主题 / 背景颜色以外的其他设置（例如字号、音效、AI 模型配置等） THEN 系统 SHALL CONTINUE TO 正确保存并在下次进入时读出这些设置，不被此次持久化修复影响
