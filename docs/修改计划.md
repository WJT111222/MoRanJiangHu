## 修复组 1: 门派商品缺失字段（3 个错误）

### 位置
`App.tsx:2093-2096`

### 错误信息
```
Property '描述' does not exist on type '门派商品'.
Property '品质' does not exist on type '门派商品'.
Property '重量' does not exist on type '门派商品'.
```

### 类型链
```
state.玩家门派: 详细门派结构 | undefined
  → sect: 详细门派结构 (after !sect guard)
  → sect.兑换列表: 门派商品[]
  → .find((g: any) => ...): 门派商品 | undefined (after !good)
  → good.描述 → ❌ 不存在
```

### 根因
`models/sect.ts:28` 的 `门派商品` 接口只有 6 个字段（`id、物品名称、类型、兑换价格、库存、要求职位`），但代码期望还有 `描述`、`品质`、`重量`。

### 修复方案
```typescript
// models/sect.ts
export interface 门派商品 {
    id: string;
    物品名称: string;
    描述?: string;         // ← 新增
    品质?: string;         // ← 新增
    重量?: number;         // ← 新增
    类型: '武学' | '丹药' | '装备' | '材料' | string;
    兑换价格: number;
    库存: number;
    要求职位: string;
}
```

### 风险
- ✅ 低。可选字段，向后兼容
- ✅ 运行时数据可能有这些字段（从其他途径填充），类型只是漏写了

### 验证
- `npm run build` 后行 2093-2096 不再报错

## 修复组 2: 兑换物品不满足 `游戏物品` 类型（2 个错误）

### 位置
`App.tsx:2103,2104`

### 错误信息
```
Argument of type '{ 物品列表: {...}[]; ... }' is not assignable to parameter of type 'SetStateAction<角色数据结构>'.
  Types of property '物品列表' are incompatible.
    Type '{ ID: string; ... 类型: string; ... }' is not assignable to type '游戏物品[]'.
      Type '{ ... 类型: string; ... }' is missing the following properties from type '消耗品': 使用效果, 毒性, 价值, 当前耐久, and 2 more.
```

### 根因（三层问题）

| 层 | 问题 | 说明 |
|----|------|------|
| 1 | `newItem` 缺少 `基础物品` 的 4 个必填字段 | `价值`, `当前耐久`, `最大耐久`, `词条列表` |
| 2 | `good.类型: string` 不能赋值给 `物品类型` | TSC 无法缩小 union 成员 |
| 3 | `good.品质: string` 不能赋值给 `物品品质` | TSC 无法缩小 union 成员 |

### 修复方案

```typescript
const newItem = {
    ID: `exchange_${goodId}_${Date.now()}`,
    名称: good.物品名称 || '兑换物品',
    描述: good.描述 || '',
    类型: (good.类型 || '消耗品') as 物品类型,           // ← as 断言（运行时是合法值）
    品质: (good.品质 || '良品') as 物品品质,             // ← as 断言（运行时是合法值）
    重量: good.重量 || 0.5,
    堆叠数量: 1,
    是否可堆叠: false,
    价值: 0,                                              // ← 新增
    当前耐久: 1,                                          // ← 新增
    最大耐久: 1,                                          // ← 新增
    词条列表: [],                                         // ← 新增
};
```

### 风险
- ✅ 低。`as 物品类型` / `as 物品品质` 是告知 TSC "运行时类型比推断更窄"，属于常规合法用法
  - `good.类型` 虽在类型系统中被展平成 `string`，但运行时一定是合法 `物品类型`（来自 AI/设定的生成规范）
- ✅ `价值: 0`、`当前耐久: 1` 等默认值合理——该游戏由 LLM 叙事驱动，物品的"真实存在感"来自故事而非数值计算。兑换物品不走数值计算逻辑，默认值不会产生实际影响

### 验证
- `npm run build` 后行 2103-2104 不再报错

## 修复组 4: 进度类型缺少 `"stream"` phase（3 个错误）

### 位置
`App.tsx:3278,3279,3281`

### 错误信息
```
Type '开局独立阶段进度 | null' is not assignable to type 'VariableGenerationProgress | null | undefined'.
  Types of property 'phase' are incompatible.
    Type '"cancelled" | "start" | "error" | "skipped" | "done" | "stream"' is not assignable to
    type '"cancelled" | "start" | "error" | "skipped" | "done"'.
      Type '"stream"' is not assignable to type '"cancelled" | "start" | "error" | "skipped" | "done"'.
// 同上 PolishProgress, MapUpdateProgress
```

### 根因
`开局独立阶段进度` 的 `phase` 包含 `'stream'`，但 `PolishProgress`、`VariableGenerationProgress`、`MapUpdateProgress` 的 `phase` 不包含。而 `WorldEvolutionProgress` 和 `PlanningProgress` 已有 `'stream'`——说明这是某次添加 `'stream'` 时遗漏了这 3 个类型。

### 修复方案

```typescript
// components/features/Chat/InputArea.tsx
type PolishProgress = {
    phase: 'start' | 'stream' | 'done' | 'error' | 'skipped' | 'cancelled';  // ← 加 stream
    // ...
};
type VariableGenerationProgress = {
    phase: 'start' | 'stream' | 'done' | 'error' | 'skipped' | 'cancelled';  // ← 加 stream
    // ...
};
type MapUpdateProgress = {
    phase: 'start' | 'stream' | 'done' | 'error' | 'skipped' | 'cancelled';  // ← 加 stream
    // ...
};
```

### 安全证明
`InputArea.tsx:134`：
```typescript
const 队列阶段运行中 = (phase?: string): boolean => phase === 'start' || phase === 'stream';
```
运行时已显式处理 `'stream'`，类型定义只是落后于运行时。

### 风险
- ✅ 低。加宽 union 是安全的类型变化，不破坏任何穷举检查
- ✅ 运行时行为不变（已处理 `'stream'`）

### 验证
- `npm run build` 后行 3278、3279、3281 不再报错
