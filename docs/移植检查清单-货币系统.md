# 货币系统移植检查清单

> 从 Rebase 分支移植到 Currency 分支（基于 origin/main）

## 背景

- **Currency 分支**基于 origin/main，main 已有自己的货币系统实现（`CurrencySystemEditor`、`NewGameCurrencySystemSetup`、`currencyDisplay.test` 等）
- **Rebase 分支**有一套独立的货币系统实现（`currencyTemplateExpander`、`currencySystemLoader`、`economyRules`、`currencyReference`）
- 两套实现**互不兼容**，不能简单合并

---

## 移植决策

### 方案 A：用 Rebase 的实现替换 main 的实现
- 删除 main 的 `CurrencySystemEditor`、`NewGameCurrencySystemSetup` 等相关文件
- 复制 Rebase 的 4 个新文件
- 修改 `models/system.ts`、`utils/apiConfig.ts` 等已有文件
- **风险**：高。main 的货币功能较多（编辑器、设置 UI、测试），替换工作量大

### 方案 B：只移植 Rebase 的新增文件，保留 main 的实现
- 复制 4 个新文件（`currencyTemplateExpander`、`currencySystemLoader`、`economyRules`、`currencyReference`）
- 修改 `models/system.ts` 新增接口
- 修改 `utils/apiConfig.ts` 新增函数
- 保留 main 的 `CurrencySystemEditor` 等 UI 不变
- **风险**：中。两套系统可能冲突（如 `获取展开货币系统` 与 main 的货币加载逻辑）

### 方案 C：先对齐 main 的货币修改，再考虑 Rebase 的新功能
- 先确保 Currency 分支包含 main 的所有货币修改
- 暂不移植 Rebase 的新功能
- **风险**：低。但 Rebase 的新功能（汇率时效、品质升级链）无法使用

**推荐方案 B**：移植 Rebase 的新增文件和接口，保留 main 的 UI 层。
