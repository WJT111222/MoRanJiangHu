# 多货币汇率系统移植方案

## 命名

Rebase 的货币逻辑应称为 **"多货币汇率系统"**（Multi-Currency Exchange System），简称 **MCX**。

它与 main 的"基础货币系统"（Base Currency System）是**两套独立但可共存**的实现。

---

## 两套系统的定位

| | 基础货币系统（main） | 多货币汇率系统（Rebase） |
|---|---|---|
| **配置格式** | `profile.economy.currencySystem` — 单体系对象 | `profile.economy.currencySystem.systems` — 多体系字典 |
| **汇率** | 静态 `baseRate` | 动态 `ExchangeRatePolicy` |
| **金钱存储** | `上层/中层/底层` + `baseAmount` | `货币桶: { systemId: { unitId: number } }` |
| **UI** | CurrencySystemEditor + NewGameCurrencySystemSetup | 无（依赖外部配置） |
| **存档兼容** | 旧存档已有 | 新存档或迁移后 |

---

## 移植方案：共存模式

### 核心思想

将 MCX 作为**额外的可选层**添加到 Currency 分支，不替换基础货币系统。通过配置中的 `systems` 字段是否存在来判断启用哪套系统。

```
profile.economy.currencySystem 结构：
{
  id: "xxx",                  // 基础货币系统（必填，兼容旧存档）
  name: "xxx",
  baseUnitId: "铜钱",
  units: [...],
  
  systems: {                  // 多货币汇率系统（可选）
    "xiuxian": { ... },
    "mortal": { ... }
  },
  exchangeRatePolicies: { ... }  // 汇率策略（可选）
}
```

### 移植步骤

#### 第 1 步：新增文件（无冲突）

```bash
git checkout Rebase -- utils/currencyTemplateExpander.ts
git checkout Rebase -- utils/currencySystemLoader.ts
git checkout Rebase -- utils/economyRules.ts
git checkout Rebase -- prompts/runtime/currencyReference.ts
```

#### 第 2 步：扩展 `models/system.ts`（增量修改）

在 `CurrencySystem` 接口中新增可选字段：

```typescript
export interface CurrencySystem {
    id: string;
    name: string;
    baseUnitId: string;
    units: CurrencyUnit[];
    formatStyle?: 'single' | 'compound';
    // === 多货币汇率系统扩展（可选）===
    systems?: Record<string, ExpandedSystemConfig>;      // 多体系共存
    exchangeRatePolicies?: ExchangeRatePolicies;          // 动态汇率策略
}
```

在文件末尾追加 8 个新接口（`QualityTemplate`、`UnitDefinition` 等）。

**冲突**：✅ 无。只是新增字段和接口，不影响 main 的现有逻辑。

#### 第 3 步：扩展 `utils/apiConfig.ts`（增量修改）

新增 3 个函数（`是否多货币模式`、`获取展开货币系统`、`重置货币系统缓存`）。

```typescript
// 是否启用多货币汇率系统
export const 是否多货币模式 = (profile: any): boolean => {
    const systems = profile?.economy?.currencySystem?.systems;
    return !!(systems && Object.keys(systems).length > 0);
};
```

**冲突**：✅ 无。main 没有这些函数。

#### 第 4 步：扩展 `utils/currencyDisplay.ts`（增量修改）

在 `构建角色金钱显示快照` 入口增加多货币分支：

```typescript
export const 构建角色金钱显示快照 = (金钱原始, 运行时配置, role) => {
    // 多货币汇率系统优先
    if (是否多货币模式(运行时配置?.modeRuntimeProfile)) {
        const 展开货币 = 获取展开货币系统(运行时配置?.modeRuntimeProfile);
        if (展开货币) {
            const snapshot = 构建多系统货币快照(金钱原始?.货币桶, 运行时配置?.modeRuntimeProfile);
            if (snapshot) return snapshot;
        }
    }
    // 回退到基础货币系统
    return 原有基础货币快照逻辑(金钱原始, 运行时配置, role);
};
```

**冲突**：✅ 无。只是新增分支，原有逻辑不变。

---

## 存档兼容

| 存档类型 | 行为 |
|----------|------|
| 旧存档（无 `systems` 字段） | 走基础货币系统逻辑，与 main 完全一致 |
| 新存档（有 `systems` 字段） | 走多货币汇率系统逻辑 |
| 旧存档迁移 | `迁移货币桶旧单位名` 自动将旧单位名映射到新 ID |
