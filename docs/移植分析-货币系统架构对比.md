# 货币系统架构对比：main vs Rebase

## 核心差异

| 维度 | main 分支 | Rebase 分支 |
|------|-----------|-------------|
| **设计理念** | 单一货币体系 + 固定汇率 | 多货币体系 + 动态汇率 |
| **货币单位** | `CurrencyUnit`（扁平列表） | `UnitDefinition`（按体系分组） |
| **汇率机制** | 静态 `baseRate`（单位换算） | 动态 `ExchangeRatePolicy`（时效+区间） |
| **品质升级** | 无 | `QualityTemplate`（下品→中品→上品） |
| **跨体系兑换** | 无 | `TransformDefinition` + `TransformBatchConfig` |
| **汇率时效** | 无（固定不变） | 有（按地图层级有过期时间） |
| **汇率生成** | 无（手动配置） | AI 驱动（变量生成阶段） |
| **UI 编辑器** | `CurrencySystemEditor` + `NewGameCurrencySystemSetup` | 无（依赖外部配置） |
| **测试覆盖** | `currencyDisplay.test`（763行）+ `auctionHouse.test`（249行） | `currencyReference.test`（215行）+ `currencySystemLoader.test`（55行）+ `economyRules.test`（38行）+ `exchangeRateIntegration.test`（252行） |

## 数据模型对比

### main 的模型

```typescript
// 扁平的单位列表，每个单位有 baseRate（相对于 baseUnit 的倍数）
interface CurrencyUnit {
    id: string;
    name: string;
    baseRate: number;        // 1 个此单位 = baseRate 个 baseUnit
    order: number;
    aliases?: string[];
}

interface CurrencySystem {
    id: string;
    name: string;
    baseUnitId: string;      // 基准单位 ID
    units: CurrencyUnit[];   // 所有单位（含品质层级）
    formatStyle?: 'single' | 'compound';
}

// 金钱存储
interface 角色金钱 {
    上层货币?: number;
    中层货币?: number;
    底层货币?: number;
    baseAmount?: number;     // 最小单位总量
}
```

**特点**：
- 一个存档只有一个 `CurrencySystem`
- 汇率 = `baseRate` 比值（静态）
- 通过 `formatStyle` 区分单层/复合显示
- 金钱存储用"上层/中层/底层"三个字段

### Rebase 的模型

```typescript
// 按体系分组的单位，支持品质升级链
interface UnitDefinition {
    displayName: string;
    displayOrder: number;
    qualityUpgradeTo?: string;    // 品质升级目标
    qualityUpgradeRatio?: number; // 升级比例
}

interface ExpandedSystemConfig {
    displayName: string;
    exchangeable?: boolean;        // 是否可兑换
    units: Record<string, UnitDefinition>;
    qualityTemplate?: QualityTemplate;  // 品质模板
    transforms?: TransformDefinition[]; // 跨体系转化
}

interface ExpandedCurrencySystem {
    systems: Record<string, ExpandedSystemConfig>;  // 多体系共存
    transforms: TransformDefinition[];
    validatedPolicies?: ExchangeRatePolicies;      // 动态汇率策略
}

// 金钱存储
interface 角色金钱 {
    货币桶?: { [systemId: string]: { [unitId: string]: number } };
}
```

**特点**：
- 一个存档可有多个货币体系（如"修仙界灵石" + "世俗金银"）
- 汇率是动态的（`ExchangeRatePolicy`），有过期时间
- AI 在变量生成阶段自动更新汇率
- 金钱存储用"货币桶"（按体系→单位）

## 功能对比

| 功能 | main | Rebase |
|------|------|--------|
| 品质升级链 | ❌ 无 | ✅ QualityTemplate（下品→中品→上品自动展开） |
| 多体系共存 | ❌ 单一体系 | ✅ 多体系（修仙/世俗/末日等） |
| 动态汇率 | ❌ 静态 baseRate | ✅ 固定/区间两种类型 |
| 汇率时效 | ❌ 永不过期 | ✅ 按地图层级有过期回合数 |
| AI 生成汇率 | ❌ 无 | ✅ 变量生成阶段自动写入 |
| 跨体系兑换 | ❌ 无 | ✅ TransformDefinition |
| 货币编辑器 UI | ✅ 完整的可视化编辑器 | ❌ 无 |
| 开局货币设置 | ✅ NewGameCurrencySystemSetup | ❌ 无 |
| 货币显示 | ✅ 三层/单层显示 | ✅ 多系统快照显示 |

## 冲突分析

### 直接冲突（不可共存）

1. **`models/system.ts` 的接口定义**
   - main 的 `CurrencySystem` 使用 `units: CurrencyUnit[]`（数组）
   - Rebase 的 `ExpandedSystemConfig` 使用 `units: Record<string, UnitDefinition>`（字典）
   - 两者字段结构不同，**不能同时存在**

2. **金钱存储格式**
   - main 用 `上层货币/中层货币/底层货币/baseAmount`
   - Rebase 用 `货币桶: { systemId: { unitId: number } }`
   - **完全不同的存储格式**

3. **`utils/currencyDisplay.ts` 的入口函数**
   - main 的 `构建角色金钱显示快照` 处理三层/单层
   - Rebase 新增 `构建多系统货币快照` 处理货币桶
   - 入口逻辑需要选择走哪条路径

4. **`utils/modeRuntimeProfile.ts`**
   - main 有完整的货币配置加载和校验逻辑
   - Rebase 用 `currencySystemLoader.ts` 替代
   - **功能重叠**

### 可共存的改动

1. **`utils/economyRules.ts`** — 全新文件，main 没有
2. **`utils/currencyTemplateExpander.ts`** — 全新文件，main 没有
3. **`utils/currencySystemLoader.ts`** — 全新文件，main 没有（但与 main 的 `modeRuntimeProfile` 功能重叠）
4. **`prompts/runtime/currencyReference.ts`** — 全新文件，main 没有
5. **`models/world.ts` 的汇率字段** — 新增字段，不影响 main 的地图逻辑
6. **`services/auctionHouse.ts` 的 null 返回** — 纯增量修改

## 设计方案评估

### main 的优势
- ✅ 完整的 UI（编辑器 + 开局设置）
- ✅ 大量测试覆盖（1000+ 行）
- ✅ 静态汇率简单稳定
- ✅ 已有玩家使用，存档格式固定

### Rebase 的优势
- ✅ 多体系共存（修仙/世俗/末日可同时存在）
- ✅ 动态汇率（AI 驱动，更有叙事感）
- ✅ 品质升级链（自动展开品质层级）
- ✅ 汇率时效（离开地点后汇率会变化）
- ✅ 跨体系兑换（灵石 ↔ 金银）

### 推荐方案：**将 Rebase 作为额外的货币逻辑层**

不替换 main 的实现，而是在其基础上**增加** Rebase 的高级功能：

1. **保留 main 的 `CurrencySystem` 接口**（兼容旧存档）
2. **新增 Rebase 的 `ExpandedCurrencySystem` 接口**（扩展）
3. **在 `apiConfig.ts` 中增加桥接函数**：
   ```typescript
   // 将 main 的 CurrencySystem 转换为 Rebase 的 ExpandedCurrencySystem
   const 转换货币系统 = (mainSystem: CurrencySystem): ExpandedSystemConfig => { ... };
   ```
4. **金钱存储兼容两种格式**：
   - 旧存档 → 读取 `上层/中层/底层` → 转换为货币桶
   - 新存档 → 直接读取 `货币桶`
5. **UI 层保留 main 的编辑器**，后端支持两种模式

**好处**：
- 旧存档完全兼容
- 新存档可使用高级功能
- main 的测试全部保留
- Rebase 的功能作为"高级货币模式"可选启用

**代价**：
- 需要写桥接层代码
- 维护两套接口定义
- 测试需要覆盖两种模式
