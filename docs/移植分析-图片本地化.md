## 八、移植策略建议

1. **先移植 P0 核心链路**：按 `apiConfig.ts` → `imageAssets.ts` → `imageGenerationDiagnostics.ts` → `imageTasks.ts` → `dbService.ts` 的顺序，从底层到上层逐步替换。
2. **DB 版本号冲突**：如果 origin/main 已经是 VERSION=4，需手动合并 store 创建逻辑。
3. **代理决策重构**：如果 origin/main 保留了旧代理逻辑，建议整体替换而非合并，避免两套决策逻辑并存。
4. **测试验证**：
   - 模拟图片本地化失败（断网/无效 URL），验证降级为远程兜底引用
   - 模拟图床上传连续失败 3 次，验证冷却；再模拟成功，验证自愈
   - 模拟 IndexedDB 配额超限，验证清理最旧资源后重试
   - 验证 web 端跨域图片经代理可正常渲染
