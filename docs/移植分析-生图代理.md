## 九、移植注意事项

1. **`?url=` 模式路径拼接**：Rebase 中 `?url=` 模式不再拼接路径（直接使用目标地址），而 main 分支仍然做拼接。移植时需确认生产 Worker 端的 `openai-image-proxy` 是否也做了对应修改，否则可能导致生产环境路径错误。

2. **删除函数的兼容性**：`判断可走OpenAI图片运行时代理` 和 `转换为运行时OpenAI图片代理端点` 在 Rebase 中被删除。如果 main 分支有其他代码调用这两个函数（如测试文件或其他服务），移植后会导致编译错误。

3. **古代行囊纠偏功能**：如果 main 仍需要此功能，需单独从 imageTasks.ts 中提取并保留。

4. **`构建OpenAI兼容模型列表候选地址`**：Rebase 中此函数被删除，替换为 ImageGenerationSettings.tsx 中的内联实现。需确认无其他文件引用。
