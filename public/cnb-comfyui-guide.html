CNB COMFYUI GUIDE

如何构建属于自己的 CNB ComfyUI 后端

客户侧操作其实很简单。注册并登录 CNB 后，Fork 指定仓库，进入工作区等待自动启动，再在 墨色江湖 bacon159 二创版 填自己的 CNB 用户名称，即可自动连接自己的 ComfyUI。

重要提示：本教程只适用于 墨色江湖 bacon159 二创版。 其它版本的 墨色江湖 bacon159 二创版 自动发现地址、CNB 用户名称匹配方式、默认工作流和服务端同步脚本可能不同，请不要混用。主入口：https://msjh.bacon159.pp.ua/；备用入口：https://msjh.bacon.de5.net/。
  1. 1

    注册并登录 CNB

    先进入 CNB 官网完成注册登录，确保账号可正常访问仓库页面与启动按钮。

  2. 2

    Fork 这个仓库

    打开仓库页面后点击 Fork，把仓库复制到自己的账号下：

    https://cnb.cool/bacon159-2026/comfyui-ql

  3. 3

    经常检查并同步上游更新

    Fork 后要时刻留意仓库页面右上角的 同步更新。如果提示上游分支有更新，说明原仓库已经修复或升级了脚本、模型清单、基础镜像配置或默认工作流，请及时点击 同步更新，把自己的 Fork 更新到最新版。

    建议每次打开 CNB 工作区前先看一眼仓库是否有可同步更新；如果刚同步过更新，请重新打开或重启工作区，让新的启动脚本和配置生效。

  4. 4

    找到自己的 CNB 用户名称

    打开 CNB 个人主页，看左侧头像下方的中文用户名称。这个名称就是 墨色江湖 bacon159 二创版 里的“CNB 用户名称”，也是后端默认上报的自动连接标识。示例图里应填写的是 培根冲冲冲。

  5. 5

    进入工作区等待自动启动

    进入自己 Fork 后的仓库页面，打开 CNB 工作区。正常情况下不需要手动执行命令，只要等待系统自动执行启动流程即可。终端里看到 [Startup] 正在启动 ComfyUI: ...，说明启动脚本已经跑到最后一步；随后如果看到 ComfyUI 8188 ready、Detected ComfyUI URL: ...、initial sync ok，就表示 ComfyUI 已经启动并把地址上报给 墨色江湖 bacon159 二创版。没有报错时，CNB 通常也会自动弹出打开 ComfyUI 的提示。

    如果终端最后停在 torch.AcceleratorError: CUDA error: uncorrectable ECC error encountered，这通常不是你操作错了，也不是模型没有下载完，而是 CNB 分配到的 GPU 节点出现硬件或驱动级 CUDA ECC 异常。建议关闭当前工作区后重新创建或重启一个 L40 工作区；如果重开后仍然同样报错，把这段报错发给左侧侧边栏里的 CodeBuddy 或 CNB 反馈，让他们更换/修复 GPU 节点。

    如果终端明确报错，才尝试手动执行下面这条命令。仍然不行时，可以问问左侧侧边栏里的 CodeBuddy，让它根据报错帮你排查。

    cd /workspace && \
npm install && \
npm run cnb:start -- "python main.py --listen 0.0.0.0 --port 8188 --enable-cors-header '*'"
  6. 6

    保持 CNB 工作区页面打开

    启动后会进入一个 CNB 的 VS Code / workspace 页面，请不要关闭这个页面，它需要一直开着用于保活。页面地址通常类似：

    https://cnb-5n8-1jo108n4p-001.cnb.space/?folder=/workspace

    如果这个页面被关闭、浏览器休眠或工作区断开，ComfyUI 后端可能会停掉，墨色江湖 bacon159 二创版 生图时就会连接失败。

  7. 7

    回到 墨色江湖 bacon159 二创版 自动发现后端

    等 CNB 环境启动完成后，回到 墨色江湖 bacon159 二创版 的“自动发现 ComfyUI 后端”，填入自己的 CNB 用户名称并点击刷新列表。用户名称匹配后会显示当前 ComfyUI 域名，并自动回填 API 地址。

  8. 8

    开启默认工作流

    回到 墨色江湖 bacon159 二创版 的文生图设置，确认 ComfyUI 后端类型已经选中，并打开 使用默认工作流。普通玩家建议优先使用默认工作流，不需要导出或上传任何 workflow JSON。

    默认工作流会跟随 墨色江湖 bacon159 二创版 更新，减少因为旧工作流、缺节点或模型名称变化导致的生图失败。

  9. 9

    可选：自定义 ComfyUI 工作流

    只有在你明确想使用自己的 ComfyUI 节点链路时，才需要关闭 使用默认工作流，然后走自定义 workflow 流程。

    在已经启动的 ComfyUI 中打开你要使用的工作流，点击左上角菜单，选择 导出（API），保存得到的 .json 文件。注意要选 API 导出，不是普通导出。

    再回到 墨色江湖 bacon159 二创版 的文生图设置，在 ComfyUI Workflow JSON 上方点击 上传 API 文件。系统会自动把常见提示词、负面提示词、尺寸和采样参数替换为占位符并填入配置。

打开墨色江湖 bacon159 二创版 打开仓库并 Fork 打开 CNB 官网
如果后端已经启动，但 墨色江湖 bacon159 二创版 里还没出现在线地址，先确认客户端填写的是 CNB 个人主页头像下方的用户名称，再点一次“刷新列表”。正常情况下，启动成功后系统会自动发现并展示匹配用户名称的 8188 ComfyUI 域名。使用期间请保持 CNB 的 VS Code / workspace 页面打开；如果页面关闭或休眠，生图可能提示连接不到服务器或跨域失败。 如果报错包含 uncorrectable ECC error encountered，优先按 CNB GPU 节点异常处理：重开工作区或联系 CNB/CodeBuddy，不需要反复重装模型。
