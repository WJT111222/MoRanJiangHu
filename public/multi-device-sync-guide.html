SAVE SYNC GUIDE

多端同步教程

这篇教程用于把手机 APK、电脑浏览器、平板或备用设备里的存档保持一致。推荐先在一台常用设备上完成云同步配置，再让其它设备登录同一个同步账号并拉取云端存档。

进入主站 进入备用站 下载最新版 APK

同步前先确认

  • 所有设备尽量更新到同一个版本。首页“发布信息”里应显示相同的 Web / APK 版本号。
  • 第一次同步前，先在原设备手动保存一次，确认“读取进度”里能看到最新存档。
  • 跨设备同步前不要同时在两台设备连续游玩同一个存档，避免后保存的一端覆盖另一端。
  • 如果存档很大，首次上传和下载会比较慢。请保持页面在前台，不要锁屏或切到后台太久。

推荐方式：对象存储同步

  1. 在原设备进入游戏首页或游戏内菜单，打开“云端同步”。
  2. 选择“对象存储同步”。填写客户提供的用户名、Access Key、Secret Key、Endpoint、Bucket 和同步前缀。
  3. 点击连接测试。通过后，先执行“上传到云端”。
  4. 在另一台设备打开同一版本的游戏，填写同一套对象存储配置。
  5. 执行“从云端恢复”或“下载云端存档”，再进入“读取进度”选择最新存档。

没有对象存储账号也可以白嫖 hi168：无需实名，免费额度约 500G，速度也不错。参考帖：NodeSeek hi168 教程。这个网址需要翻墙才能打开。

备用方式：GitHub 云同步

  1. 打开“云端同步”，选择 GitHub 同步。
  2. 按页面提示登录 GitHub 并完成授权。
  3. 在原设备点击上传云端存档。
  4. 在新设备登录同一个 GitHub 账号，然后从云端恢复。

GitHub 方式适合没有对象存储账号的玩家；对象存储方式更适合频繁同步和较大的存档。

日常使用建议

从 A 设备换到 B 设备

  1. 在 A 设备先保存游戏。
  2. 打开云端同步，上传最新存档。
  3. 在 B 设备打开云端同步，下载或恢复云端存档。
  4. 进入“读取进度”，选择刚同步过来的最新存档。

从 B 设备玩完再回 A 设备

  1. 在 B 设备保存并上传。
  2. 回到 A 设备后先下载或恢复云端存档，再读取。
  3. 确认当前剧情、角色、背包和图片都正确后再继续游玩。

常见问题

云端没有看到最新存档

先确认原设备已经手动保存，再重新上传。上传过程中不要关闭页面。

恢复后头像或图片显示慢

大型存档首次恢复后会陆续读取图片资源。等待一会儿，或重新打开对应界面让图片缓存加载。

提示签名失败或 403

检查对象存储的 Access Key、Secret Key、Endpoint、Bucket 是否完全一致。复制时不要带空格。

安全提醒

Access Key 和 Secret Key 只填写在自己的设备里，不要发到聊天群、截图、公开工单或仓库文件。需要客服协助时，可以只提供错误提示和同步方式，不要提供密钥。

如果怀疑密钥泄露，请先到对象存储控制台禁用或重置密钥，再在所有设备上更新新密钥。
